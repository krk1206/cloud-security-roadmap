# Network Basics & Security Summary

## 학습 개요
- 네트워크관리사 2급 특강 대비를 위해
  네트워크 기초 개념과 보안 관점을 중심으로 학습함
- 암기보다는 전체 구조와 흐름 이해를 목표로 정리

---

## 핵심 개념 요약

### 네트워크 식별자
- **MAC Address**: 같은 네트워크 대역 내 장비 식별
- **IP Address**: 네트워크 간 통신을 위한 논리적 주소
- **Port Number**: 하나의 IP에서 실행 중인 서비스 식별자

---

### TCP vs UDP
- TCP
  - 연결형, 신뢰성 중심
  - 순서 보장, 수신 확인(ACK)
  - 웹, 파일 전송 등 정확성이 중요한 서비스에 사용
- UDP
  - 비연결형, 속도 중심
  - 순서/수신 확인 없음
  - 스트리밍, 게임 등 실시간성이 중요한 서비스에 사용

---

### OSI 7 Layer 개요
- 네트워크 문제를 계층별로 나눠 분석하기 위한 참조 모델

| 계층 | 역할 요약 |
|----|----|
| Application(7) | 사용자와 직접 연결되는 서비스 |
| Transport(4) | 종단 간 통신, TCP/UDP |
| Network(3) | IP 기반 라우팅 |
| Data Link(2) | MAC 기반 전송, 프레임 |
| Physical(1) | 물리적 신호 전송 |

※ 실무에서는 OSI 5~7 계층을 Application으로 묶어 이해

---

### TCP/IP 모델
- 실제 네트워크 통신에서 사용되는 구조
- OSI 모델을 단순화한 형태

| TCP/IP 계층 | 역할 |
|------------|------|
| Application | 사용자 서비스 |
| Transport | TCP / UDP |
| Internet | IP, Routing |
| Network Access | MAC, Physical |

---

## 리눅스 네트워크 & 포트
- 포트는 서버에서 실행 중인 서비스의 출입구
- 열린 포트는 외부 접근 가능 지점
- ufw를 이용해 포트 허용/차단 실습 진행
- 불필요한 포트 개방은 공격 표면 증가로 이어짐

---

## 네트워크 보안 기초

### 포트 스캔
- 서버에 열려 있는 포트를 탐색하는 행위
- 공격자는 항상 포트 스캔 후 공격을 시도

### SYN Flood
- TCP 3-way handshake 구조를 악용
- 연결을 완료하지 않아 서버 자원 고갈 유도

### UDP 증폭 공격
- UDP의 비연결성을 이용
- 작은 요청으로 큰 응답 트래픽 발생
- DNS, NTP 서비스가 자주 악용됨

### 방화벽
- IP, 포트, 프로토콜 기준으로 트래픽 제어
- 서버 보안의 1차 방어선
- 리눅스 ufw ↔ 클라우드 보안 그룹과 동일한 개념

---

## 보안 관점 핵심 정리
- 공격 흐름: 포트 스캔 → 취약점 악용
- TCP는 세션 기반 공격,
  UDP는 반사/증폭 공격에 취약
- 최소 포트 허용이 보안의 기본 원칙

---

## 한 줄 정리
네트워크는 구조를 이해해야 보안이 보이고,  
보안은 불필요한 노출을 줄이는 것에서 시작된다.
